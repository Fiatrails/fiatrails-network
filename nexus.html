Nexus

Agentic AI settlement. Autonomous clearing agents operating over causal shared state with graph-derived credit pricing and real-time settlement optimisation.

← Back to Products
Concept

AI-native clearing

Current clearing systems execute deterministic rules. Nexus introduces autonomous AI agents that observe the full settlement graph, identify optimisation opportunities humans cannot see, and execute clearing decisions in real time. Each agent operates over causal shared state — a unified view of all obligations, positions, and counterparty relationships across the Settlement Computer.

Planned Capabilities

What Nexus will deliver

01

Causal shared state

A unified, causally consistent view of all obligations, net positions, and counterparty relationships across every corridor and product. Agents read from the same state without conflicts or stale data.

02

Graph-derived credit pricing

Real-time credit risk assessment derived from the obligation graph topology. Agents identify concentration risk, counterparty exposure chains, and systemic vulnerabilities that flat models miss.

03

Autonomous netting optimisation

AI agents continuously evaluate netting window timing, participant grouping, and corridor sequencing to maximise compression. Dynamic window sizing based on real-time obligation flow patterns.

04

Predictive liquidity management

Agents forecast net settlement positions before window close, enabling participants to pre-position liquidity. Reduces settlement failures caused by insufficient funding at settlement time.

05

Cross-product orchestration

Nexus agents operate across Settlement, PayNet, RiskNet, Trade, and Grid simultaneously. A single agent can identify that a Trade obligation offsets a Settlement position and propose cross-product netting.

06

Anomaly detection

Continuous monitoring of obligation patterns for anomalous behaviour — unusual volumes, atypical counterparty combinations, timing irregularities. Flagged to corridor administrators with causal explanations.

Architecture

Agent architecture

Agent types

Nexus deploys specialised agents for different clearing functions, each with defined observation scope and action authority.

  • Netting optimisation agents
  • Credit assessment agents
  • Liquidity forecasting agents
  • Anomaly detection agents
  • Cross-product orchestration agents

Safety constraints

All agent actions are bounded by configurable safety constraints. No agent can execute a settlement action that violates corridor rules or participant limits.

  • Action authority scoped per agent type
  • Human-in-the-loop for high-value decisions
  • Full audit trail of agent reasoning
  • Corridor admin override at any time
  • Deterministic fallback if agents disagree
FX Pricing

Oracle-determined FX rates

All Settlement Computer products use FX rate feeds from AFXO — an institutional-grade FX oracle delivering deterministic rate feeds across global currency pairs. Nexus agents consume real-time oracle data for cross-currency position analysis and liquidity forecasting.
